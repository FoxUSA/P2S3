Sign URL

Create a S3 signed url using the getSignedUrl command

Upload to URL

Upload a file to a signed url using a HTTP PUT

About

This utility security generate a URL. You can send someone this link which allows them to upload a file to S3 without requiring you to creating them an account or give them credentials. This method uses the getSignedUrl method provided by AWS.

Once you generate the URL, ether send the signed url and a link to this app. Or send a single combined link.

The max file size is 5 GB.

Usecases

You are a consultant and need to have a way of uploading a file securely to a clients account. If your client didn't provide you with something like DropBox.

Want to upload a file from another PC and but do not want to log into AWS account or transfer your keys to the machine.

Security

This utility does not save your credentials. Your browser is communicating with AWS directly. This utility is also open source so the community can audit the code.

The url generated only allows a specific file type, you specify the location it will be uploaded, and will expire at a configured time. If you enable encryption at link creation, we will tell S3 to encrypt the object server side with AES 256 using the default key in AWS KMS.
